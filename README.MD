# Развёртывание образа из GitHub Container Registry на общий VPS

Краткое описание: как образ приложения **gitA** (сборка и публикация в **GHCR** через GitHub Actions) был развёрнут на том же сервере, где уже работал другой проект.

## Контекст

На виртуальной машине уже был развёрнут стек **registry-autodeploy**:

- приватный **Docker Registry** на порту **5000** (HTTPS);
- **Watchtower**;
- тестовое **FastAPI-приложение** (autodeploy) на порту **8000** с отдельным `docker-compose` из каталога `fastapi-app/`.

Проект **gitA** ([репозиторий на GitHub](https://github.com/natashakalash-tech/gitA)) публикует образ в **GitHub Container Registry** (`ghcr.io`), а не в локальный Registry. Поэтому для его запуска на сервере **не требовался** `docker login` к адресу `194.67.99.85:5000` — достаточно публичного образа (или отдельной авторизации в `ghcr.io`, если пакет приватный).

## Избежание конфликта портов

| Сервис              | Порт на хосте |
|---------------------|---------------|
| Приватный Registry  | 5000          |
| Приложение autodeploy | 8000        |
| **gitA** (To-Do API) | **8080**     |

Порт **8080** выбран так, чтобы **не пересекаться** с уже занятыми **5000** и **8000**.

## Что лежит в этом каталоге

- **`docker-compose.yml`** — тянет готовый образ `ghcr.io/natashakalash-tech/gita:latest` и пробрасывает **8080:8080**.

## Шаги развёртывания на сервере

1. На сервер доставлен каталог **`gitA-server`** (или весь репозиторий **registry-autodeploy**), например через `git clone` с GitHub и переход в `.../gitA-server`.
2. В каталоге `gitA-server` выполнено:

   ```bash
   docker compose pull
   docker compose up -d
   ```

3. Образ предварительно появляется в GHCR после успешного workflow **Deploy** при push в ветку **main** (вкладка **Actions** в репозитории **gitA**).

## Проверка работы

В браузере: **`http://<IP-сервера>:8080/docs`** — интерфейс Swagger для FastAPI (эндпоинты `/tasks` и др.).

При необходимости на сервере открыт входящий TCP **8080** в фаерволе (`ufw` или панель хостинга).

## Остановка сервиса

Чтобы освободить порт и не держать контейнер запущенным:

```bash
cd /путь/к/gitA-server
docker compose stop
```

или полное удаление контейнера проекта:

```bash
docker compose down
```

Остальные compose-проекты на той же машине при этом **не затрагиваются**, если останавливать только этот файл в каталоге `gitA-server`.

## Связь с CI/CD

- Сборка образа и `docker push` в **GHCR** выполняются **на раннерах GitHub Actions** (файл `.github/workflows/deploy.yml` в репозитории **gitA**).
- На **VPS** выполняется только **pull готового образа** и **запуск контейнера** через `docker compose` — классическое разделение: CI собирает и публикует, сервер потребляет артефакт из реестра.
